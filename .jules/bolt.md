## 2024-11-20 - Deep Simulation Loop Optimization
**Learning:** In a matrix-based game tree like Chain Reaction, high-frequency deep engine cloning inside the Minimax lookahead search can be a massive performance bottleneck. Specifically, using JS/TS high-level collection features such as `Array.prototype.map()` and dynamic object spreading `{ ...cell }` inside recursive steps creates significant garbage collection and call stack overhead. Pre-allocating standard arrays and using raw `for` loops along with lazy object allocation (e.g. bypassing grid initialization when the grid is immediately overwritten) reduces calculation latency by over 70%.
**Action:** Always prefer raw `for` loops, pre-allocated fixed-size arrays, and bypassed/lazy object initializers in any high-frequency simulation, matrix cloning, or search trees.
